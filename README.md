# portfolio
2026.02.23〜　RUNTEQ卒業制作

# 1. サービス概要

仕事と学習の締切に追われる社会人が、
“今日やること”を迷わず決められるタスク管理サービスです。
未完了タスクを締切が近い順で表示することで、判断疲れを減らし、
先延ばしや期限切れを防ぎます。
• 誰の、どんな課題を、どう解決するか
•	誰の：仕事と学習を両立して忙しい社会人
•	課題：タスクが多いほど「何からやるか」を決めるだけで疲れてしまい、行動を始められない
•	解決：未完了×締切が近い順で表示し、最初の一歩を即決できるようにする

⸻

# 2. このアイデアはどこから生まれたか

**2-1. きっかけとなった体験・感情**

仕事をしながらRUNTEQのカリキュラムを進めていた期間、
平日は仕事と残業で疲れ切ってしまい、思うように学習時間が取れず、
その分を休日にまとめて進めようとしていました。
仕事の締切と学習の課題が同時に迫る中で、
「今日は何から手を付ければいいんだろう」と考えるだけでしんどくなり、
気づくとスマホを見て時間が過ぎていることもありました。

やるべきことは分かっているのに、
タイムマネジメントができない・先を見通せない自分に対して、
焦りや自己嫌悪を感じることも多くありました。

**2-2. なぜそれが「気になった」のか**

タイムマネジメントが苦手でやるべきことが多いほど
抱え込んでしまう性格のため、
タスクを頭の中だけで管理しようとして限界を感じていました。
忙しい時ほど、落ち着いて前に進める状態を作りたいと考えており、
「タスクを増やす仕組み」ではなく
判断の手間を減らして最初の一歩を軽くする仕組みが必要だと感じました。

⸻

# 3. 課題の整理

**3-1. 表に見えている困りごと**
•	タスクが増えると、何がいつまでか把握できなくなる
•	優先順位がつけられず、重要なものを後回しにしてしまう
•	仕事後は疲れて、考えること自体がしんどい

**3-2. 本当に解決したい課題**
•	なぜ？：タスクが多いと、優先順位の判断が難しい
•	なぜ？：判断するには、締切・重要度・作業量などを同時に考える必要がある
•	なぜ？：疲れているときほど判断ができず、手が止まって先延ばしになる

→ 本当に解決したい課題：
忙しい日ほど判断疲れで行動できなくなり、やるべきことが進まない状態

⸻

# 4. 想定ユーザー

**4-1. 想定しているユーザー**
•	年齢：23〜40歳
•	生活・状況：仕事をしながら、学習・副業・資格勉強などにも取り組んでいる
•	環境：スマホ中心。移動中や仕事後のスキマ時間に確認・更新したい

**4-2. 自分とユーザーの距離**
	•	自分自身が実際に同じ課題に直面しており、困りごとの解像度が高い

**4-3. 実際に使われる可能性**
•	使ってくれそうな人：自分、同じように仕事と学習を両立している友人
•	使う場面：仕事後に「まだ終わっていないタスク」を整理したいとき
•	代替手段：メモ、カレンダー、Notion、Todoist
•	置き換える理由：未完了×締切順で“判断を減らせる”体験に特化しているため

**4-4. 導入・継続のイメージ**
•	きっかけ：タスクが溜まって混乱したとき
•	最初の一歩：ログイン後、タイトルと締切日だけでタスク登録
•	継続理由：開くだけで「今やるべき順」が分かる、完了チェックで達成感が得られる

⸻

# 5. 既存サービス・競合

**5-1. 似たサービス**
•	サービス名：Todoist
•	URL：https://todoist.com/
•	できること：タスク管理、プロジェクト分け、優先度設定など

**5-2. それでも作りたい理由**

高機能なToDoアプリは自由度が高い分、
忙しい時ほど設定や整理の手間が増えてしまいます。
本当に欲しかったのは、機能の多さではなく、
**締切前に迷わず着手できる“シンプルな一覧体験”**でした。

**5-3. 差別化ポイント**

仕事と学習を両立する忙しい社会人が、
未完了タスクだけを締切が近い順で見て、迷わず着手できる体験に特化しています。

⸻

# 6. このサービスで提供したい価値
•	行動の変化：判断に悩む時間が減り、着手が早くなる
•	気持ちの変化：焦りや自己嫌悪が減り、落ち着いて取り組める
•	考え方の変化：「気合い」ではなく「仕組み」で前に進めるようになる

このサービスは、仕事と学習で忙しい人が、
締切タスクを整理して“今日の一手”を迷わず決められるようになるサービスです。

⸻

# 7. このアプリで実現すること

MVPで作る機能
•	ユーザー登録 / ログイン
•	タスク登録（タイトル、締切日、メモ）
•	タスク一覧（未完了のみ表示、締切が近い順）
•	タスク詳細
•	タスク編集 / 削除
•	完了 / 未完了の切り替え
•	「今日やるべきこと」の提示（未完了のうち、締め切りが最短のタスク表示）

本リリースで作る機能
**判断をさらに減らす改善**
•	残日数の表示（例：あと3日）
•	期限が近いタスクの強調表示
•	週間の見通し表示

⸻

# 8. 懸念点と対策
•	問題になりそうな点：既存のToDoアプリから乗り換える理由が弱い可能性
•	なぜそう思うか：競合が多く、機能で勝負すると埋もれやすい
•	対策：機能ではなく「未完了×締切順の一覧体験」に価値を集中させる、まずは自分＋身近な人に使ってもらい、改善を重ねる

⸻

# 9. 今後の展開
•	機能追加：カテゴリ、優先度、期限アラート
•	UI改善：危険度ハイライト
•	体験面の工夫：入力項目を最小限にし、登録のハードルを下げる

⸻

# 10. 技術スタック

**10-1. 使用予定の技術**
•	フレームワーク：Ruby on Rails
•	DB：PostgreSQL
•	デプロイ先：Render
•	バージョン管理：GitHub
•	CSSフレームワーク：Bootstrap
•	使用予定ライブラリ：Devise、RuboCop
• 開発環境：Docker

**10-2. 技術選定の理由**
•	Rails：CRUD＋認証を最短で実装でき、MVP構築に向いている
•	PostgreSQL＋Render：構築が安定しており、詰まりにくい
•	GitHub：Issue/PRで開発プロセスを可視化できる
•	Bootstrap：UIを短時間で整え、機能実装に集中できる
•	Docker：環境差を減らすため
